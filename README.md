# Towers

Настольная игра на соединение флагов на поле 10×10 — в духа Hex/Bridg-It, но со звёздами и стенами. Играйте вдвоём за одним экраном или против сильного ИИ, которого крайне трудно обыграть.

Вся игра — один файл [`game.html`](game.html). Никаких зависимостей, сборки и сервера: просто откройте его в браузере.

## Быстрый старт

```
Откройте game.html в любом современном браузере
```

Или запустите локальный сервер и зайдите на `http://127.0.0.1:5500/game.html` (например, через расширение Live Server в VS Code).

## Цель

Первым построить непрерывную цепочку из своих фигур, соединяющую **ваш** флаг с флагом **соперника**:

- 🔴 Красный соединяет свой флаг с синим.
- 🔵 Синий соединяет свой флаг с красным.

Флаги стоят в противоположных углах поля.

## Как играть

- Игроки ходят по очереди. За ход вы ставите **одну** фигуру на любую свободную клетку (стена занимает 3 клетки сразу).
- Любую пустую клетку может занять любой игрок — поле общее.
- Игра заканчивается, как только один из игроков замыкает цепочку между флагами.

### Соединение

- Свои фигуры связаны, если стоят рядом **по горизонтали или вертикали**.
- По **диагонали** связь работает только через **звезду**: хотя бы одна из двух соседних по диагонали клеток должна быть звездой.
- Победа засчитывается, когда непрерывная цепочка своих фигур тянется от одного флага до другого.

### Фигуры

| Иконка | Название | Описание |
|--------|----------|----------|
| 🟢 | **Фигура** | Обычный элемент. Ставится без ограничений. |
| ⭐ | **Звезда** | 1 за партию. Даёт диагональные связи — путь может идти «наискосок». |
| 🧱 | **Стена** | 1 за партию, занимает 3 клетки в ряд. Блокирует соперника. Клавиша **R** — повернуть стену. Свой путь стеной перекрывать нельзя. |

## Режимы

- **👥 1 на 1** — двое игроков за одним экраном.
- **🧠 ИИ Perfect Engine** — вы играете красным против ИИ.

## Об ИИ (Perfect Engine)

ИИ рассчитан на то, чтобы его было почти невозможно обыграть. Ключевые принципы движка:

- **Гонка соединений.** Позиция оценивается через `connectionCost` — минимальное число фигур, которые каждому игроку нужно доложить, чтобы соединить флаги (с учётом диагоналей через звёзды). ИИ стремится соединиться первым.
- **Поиск в глубину.** Минимакс с альфа-бета отсечением и итеративным углублением под бюджет времени — ИИ просчитывает партию на несколько ходов вперёд, а не жадно на один.
- **Защита пути.** `pathVulnerability` штрафует ИИ за «узкий» путь с бутылочными горлышками и поощряет держать несколько маршрутов — так соперник не может его запереть, отрезав единственную линию.
- **Форсированные тактики.** ИИ немедленно завершает своё соединение, если может, и обязательно блокирует соперника, когда тот в одном ходу от победы.

> **Честная оговорка.** Поле в Towers общее (оба игрока делят одни клетки и могут вставать на путь друг друга), поэтому математически гарантированной «непобедимости» здесь не существует. При идеальной игре соперника партия сводится вничью; при любой неточности ИИ выигрывает.

## Технологии

Чистые HTML, CSS и JavaScript в одном файле. Без библиотек и внешних ресурсов.

## Структура файла

Вся логика в `game.html`:

- **Поиск путей:** `connectionCost`, `getPathDistance`, `getShortestPathMissingCells`, `checkWin` — BFS по полю с учётом фигур, звёзд и стен.
- **ИИ:** `evaluate`, `pathVulnerability`, `minimax`, `makeAIMove` — оценка позиции и выбор хода.
- **Игра/UI:** `build`, `onClick`, `finishTurn`, `renderCell`, `updateStatus` — состояние партии и отрисовка.
